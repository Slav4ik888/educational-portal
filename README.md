
1. Пошаговый план разработки:

День 1-2: Настройка Webpack + TS + FSD структура + базовый роутинг

День 3: Создай 1-2 мок-статьи в JSON и научись отображать теорию

День 4: Добавь тестовые блоки между главами (локальный state)

День 5: Подключи Redux для хранения ответов и прогресса

День 6: Реализуй итоговый тест в конце статьи

День 7: Прогресс-бар и сохранение прогресса в localStorage



2. Ключевые особенности, которые нужно учесть
Сохранение прогресса — используй RTK + localStorage middleware

Адаптивность — начни с десктопа, затем мобилка

Типизация — все строго типизируй с первого дня

Тестовые блоки — можно сделать разные типы:单选, множественный выбор, соответствие



3. Быстрый старт (код для копирования)
Создай файл src/shared/api/mockArticles.ts с примером статьи и сразу начинай рендерить.

Главный совет: Не пытайся сделать всё сразу. Сначала просто выведи текст статьи, потом добавь один тестовый блок в середине (локально, без Redux), затем подключай хранилище и уже потом — итоговый тест и прогресс-бар.
