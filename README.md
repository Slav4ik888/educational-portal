# 📚 Образовательный портал "Как работают LLM"

Интерактивное React-приложение для изучения больших языковых моделей (LLM) с встроенными тестами и глоссарием.

## 🚀 Демо

👉 **Попробовать онлайн:** [https://slav4ik888.github.io/educational-portal](https://slav4ik888.github.io/educational-portal)

## 🎯 О проекте

Это образовательный портал, который знакомит пользователей с принципами работы LLM на интуитивном уровне, без сложной математики. Проект объединяет теоретический материал с практическими тестами, позволяя закрепить знания прямо в процессе чтения.

**Ключевая особенность**: статья разбита на блоки с промежуточными тестами, а финальное тестирование становится доступным только после успешного прохождения всех предыдущих.

## 📦 Компоненты

### 📝 RichTextRenderer
Универсальный рендерер markdown-текста. Поддерживает:
- Заголовки (`# H1`, `## H2`, `### H3`)
- Цитаты (`> текст`)
- Списки (маркированные и нумерованные)
- Жирное выделение (`**текст**`)

### 🧪 Тестовые компоненты

| Компонент | Тип вопроса | Описание |
|-----------|-------------|----------|
| `MultipleChoice` | Одиночный/множественный выбор | Классические тесты с чекбоксами или радиокнопками |
| `TrueFalse` | Верно/Неверно | Две кнопки с визуальной обратной связью |
| `FillBlank` | Заполнение пропусков | Интерактивные поля ввода с проверкой вариантов |
| `MatchPairs` | Сопоставление пар | Drag-and-drop для соединения терминов с определениями |
| `OrderSteps` | Упорядочивание шагов | Перетаскивание элементов в правильной последовательности |

### 🔍 GlossaryText (уникальная разработка)
Кастомный компонент для встроенного глоссария:
- Автоматически находит термины в тексте
- Подсвечивает их пунктирным подчеркиванием
- При наведении показывает всплывающий tooltip с определением
- Тултип следует за курсором и адаптируется под размер экрана

## 🎮 Как пользоваться

### 1. Открытие статьи
После запуска приложения вы увидите главную страницу со статьями доступными для изучения.

### 2. Чтение и тестирование
- Статья разбита на тематические блоки
- После каждого теоретического блока следует промежуточный тест
- Для перехода к следующему блоку нужно ответить на **все вопросы** текущего теста

### 3. Глоссарий
- Термины, выделенные пунктирным подчеркиванием, имеют всплывающие определения
- Наведите курсор на любой термин (например, **LLM**, **Токен**, **Attention**)
- Появится tooltip с объяснением термина

### 4. Типы тестовых заданий

| Тип | Что делать |
|-----|------------|
| 🔘 Multiple Choice | Выберите один или несколько правильных вариантов |
| ✓ True/False | Нажмите "Верно" или "Неверно" |
| 📝 Fill Blank | Введите пропущенные слова в тексте |
| 🔗 Match Pairs | Перетащите термины к их определениям |
| 📊 Order Steps | Расставьте шаги в правильном порядке (перетаскиванием или кнопками) |

### 5. Финальный тест
- Появляется **только после успешного прохождения всех промежуточных тестов**
- Содержит 12 вопросов всех типов
- Результат показывает общий балл и процент правильных ответов

### 6. Режим разработки (опционально)
Для автоматического заполнения ответов (удобно при разработке):
```typescript
// В компоненте используется хук useDevAnswers
// При NODE_ENV === 'development' ответы заполняются автоматически
```

## 🏗 Архитектура проекта (FSD)

```
src/
├── app/          # Инициализация, провайдеры, глобальные стили
├── pages/        # Страницы приложения
├── widgets/      # Виджеты (ArticleReader, TestBlock)
├── features/     # Фичи (test, glossary)
├── entities/     # Сущности (article, question, glossary-term)
└── shared/       # Переиспользуемый код (ui, lib, config)
```

## 📁 Структура ключевых компонентов

```
shared/ui/RichTextRenderer/     # Рендерер markdown
features/test/ui/               # Компоненты тестов
  ├── MultipleChoice/
  ├── TrueFalse/
  ├── FillBlank/
  ├── MatchPairs/
  └── OrderSteps/
features/glossary/              # Глоссарий
  ├── ui/GlossaryText/         # Обертка с подсветкой
  ├── lib/highlightTerms.ts    # Функция замены терминов
  └── types.ts
entities/glossary/config/       # Словарь терминов LLM
```
