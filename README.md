# Knowledge Journey — AI-Powered Learning System

Интерактивная образовательная система с ИИ-генерацией персонализированных учебных маршрутов, геймификацией и автоматической проверкой ответов.

---

## О проекте

**Knowledge Journey** превращает любую тему или текст в структурированный учебный маршрут («путешествие»). ИИ разбивает материал на чекпоинты, подбирает задания разных типов и оценивает развёрнутые ответы студента в реальном времени. Каждое прохождение заканчивается подробным отчётом с ранжированием, историей ответов и рекомендациями для повторения.

**Ключевые возможности:**
- Генерация учебного маршрута по свободной теме или вставленному тексту
- 8 типов заданий (выбор ответа, заполни пропуски, развёрнутый ответ, «объясни другу», и др.)
- Таймер на каждый чекпоинт с автозавершением и штрафом за превышение
- Геймификация: XP, серии правильных ответов, ×2 бонус за скорость, 3 вида достижений
- AI-оценка свободных ответов с подробным фидбеком (сильные стороны / точки роста)
- Финальный отчёт с точностью по каждому чекпоинту и рекомендациями для повторения
- Печать / сохранение отчёта как PDF через браузер

## Технологии и мотивация выбора

| Технология | Роль | Мотивация |
|---|---|---|
| **React 19 + TypeScript** | UI + типизация | Актуальный стек, строгая типизация устраняет целый класс ошибок |
| **Vite** | Сборщик | Мгновенный HMR, быстрая dev-сборка даже на больших проектах |
| **Redux Toolkit** | Глобальное состояние | Предсказуемое единое хранилище; RTK устраняет boilerplate |
| **FSD (Feature-Sliced Design)** | Архитектура | Чёткие правила импортов между слоями, масштабируется без хаоса |
| **SCSS Modules** | Стили | Изолированные классы, поддержка print-CSS, нет конфликтов |
| **OpenRouter API (`openai/gpt-4o-mini`)** | ИИ-движок | Экономичная модель с высоким качеством для задач генерации и оценки |
| **Node.js HTTP-сервер** | Backend proxy | Скрывает API-ключ от клиента, реализует SSE-стриминг ответа |

## Архитектура системы (FSD)

```
src/
├── app/           # Инициализация: провайдеры, роутер, глобальные стили
├── pages/         # Страницы: JourneyNewPage, JourneyPage, JourneyReportPage, ArticlesPage
├── widgets/       # Виджеты: ArticleReader, TestBlock
├── features/      # Фичи: test, glossary
├── entities/      # Сущности:
│   ├── journey/   #   Journey + Checkpoint + Activity + ActivityAnswer (+ checkActivityCorrect)
│   ├── gamification/  # XP, streak, achievements (Redux slice + types)
│   ├── article/   #   Статьи и блоки контента
│   ├── user-progress/ # Прогресс по статьям
│   └── test-block/    # Рендер вопросов, AI-evaluation UI
└── shared/        # Переиспользуемый код:
    ├── lib/ai/    #   evaluateAnswer() — вызов AI через backend proxy
    ├── lib/hooks/ #   useCheckpointTimer — таймер с фазами (normal/warning/danger)
    └── api/       #   axios instance
```

### Поток данных

```
JourneyNewPage
  → POST /api/journey/generate  (backend → OpenRouter)
  → Redux: journey.setJourney()

JourneyPage
  → Per-checkpoint: timer + activities + AI eval
  → Redux: journey.setActivityAnswer / setAiEvaluation / completeCheckpoint
  → Redux: gamification.addXP / incrementStreak / unlockAchievement

JourneyReportPage
  ← Redux: journey.answers + progress, gamification.sessionXP / maxStreak / achievements
  → window.print()  →  PDF через браузер
```

### Генерация → Прохождение → Оценка → Отчёт

1. **Генерация**: пользователь вводит тему или текст → backend вызывает GPT-4o-mini, который возвращает JSON с чекпоинтами и заданиями
2. **Прохождение**: каждый чекпоинт имеет таймер 3–5 мин; при истечении — автозавершение с 50% XP
3. **Оценка**: детерминированные задания проверяются мгновенно; AI-задания отправляются в `evaluateAnswer()` → получают score 0–100 + фидбек
4. **Отчёт**: страница `/journey/:id/report` агрегирует данные из Redux, показывает ранг, достижения, ответы и рекомендации

## Быстрый старт

### Требования
- Node.js 18+
- npm 9+ или yarn

### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/educational-portal.git
cd educational-portal

# 2. Установите зависимости
npm install

# 3. Добавьте API-ключ в .env (или в переменные окружения Replit)
echo "OPENAI_API_KEY=sk-..." > .env
```

### Запуск (два варианта)

**Вариант A — Vite** (используется в Replit, быстрее HMR):

```bash
# Терминал 1 — backend (порт 7575)
node server.js

# Терминал 2 — Vite dev-сервер (порт 5000)
npx vite --host 0.0.0.0 --port 5000
```

**Вариант B — Webpack** (канонические скрипты из `package.json`):

```bash
# Терминал 1 — backend
node server.js

# Терминал 2 — Webpack dev-сервер
npm run dev     # → webpack serve на порту 5000
```

Откройте [http://localhost:5000](http://localhost:5000).

### Production-сборка

```bash
npm run build:prod     # Webpack → dist/
node server.js         # backend на порту 7575
# Раздайте dist/ через nginx или любой статик-сервер
```

## Пример сценария использования

### Шаг 1 — Ввод темы

Перейдите на `/journey/new`. Введите тему, например:

> **"Как работает градиентный спуск в машинном обучении"**

Нажмите **"Сгенерировать путешествие"**.

### Шаг 2 — Генерация (10–15 сек)

ИИ создаёт 4–6 чекпоинтов. Пример структуры:
- Чекпоинт 1: «Функция потерь» — 3 задания
- Чекпоинт 2: «Производная и градиент» — 4 задания
- Чекпоинт 3: «Скорость обучения (learning rate)» — 3 задания
- ...

### Шаг 3 — Прохождение

На каждом чекпоинте:
- Таймер отсчитывает 3–5 минут (зависит от количества заданий)
- Ответьте на задания: выбор ответа, «объясни другу», «найди ошибку», и др.
- Нажмите **"Проверить ответы"** — получите XP и продолжите
- Если таймер истёк — автоматический переход с 50% XP

### Шаг 4 — Финальный отчёт

После последнего чекпоинта открывается `/journey/:id/report`:

```
Результат: 84%  Ранг: Мастер 🎓
XP заработано: 320  Макс. серия: 6

Достижения: ⚡ Молниеносный (чекпоинт за 55 сек)

Рекомендуем повторить:
  32% — «Скорость обучения» (были ошибки)

[🖨️ Распечатать / Сохранить PDF]
[Начать новое путешествие →]
```
