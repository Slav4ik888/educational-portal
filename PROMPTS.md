# PROMPTS.md — Журнал промптов

Все AI-промпты, используемые в платформе. Исходный код промптов хранится в `ai-prompts.cjs`.

---

## 1. Генерация учебного маршрута (Journey)

**Файл:** `ai-prompts.cjs` → `JOURNEY_GENERATION_PROMPT`
**Эндпоинт:** `POST /api/journey/generate`
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Генерирует структурированную учебную статью по заданной теме или тексту. Возвращает JSON с заголовком, описанием и массивом чекпоинтов с объяснениями.

**Ключевые решения:**
- 3–7 чекпоинтов: нижняя граница гарантирует содержательность, верхняя — не перегружает
- `explanation` каждого раздела — минимум 3 абзаца: обеспечивает достаточно контекста для заданий
- Указание «от базового к сложному» — делает маршрут обучаемым, а не хаотичным

---

## 2. Генерация заданий (Activities)

**Файл:** `ai-prompts.cjs` → `ACTIVITIES_GENERATION_PROMPT`
**Эндпоинт:** `POST /api/journey/generate` (отдельный вызов на каждый чекпоинт)
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Создаёт 4–5 разнообразных заданий для каждого чекпоинта. Поддерживает 8 типов:
- `multiple-choice` — выбор ответа
- `true-false` — верно/неверно
- `fill-blank` — заполни пропуски (с падежными формами в `alternatives`)
- `free-response` — развёрнутый ответ (AI-оценка)
- `explain-like-im-five` — объясни ребёнку (AI-оценка)
- `teach-back` — объясни другу (AI-оценка)
- `give-your-example` — придумай пример (AI-оценка)
- `debug-the-logic` — найди логическую ошибку (AI-оценка)

**Ключевые решения:**
- `alternatives` для `fill-blank` содержит все падежные формы — решает проблему ложных ошибок
- Требование «минимум 1 AI-оцениваемый тип» — обеспечивает глубину обратной связи
- `points`: 10 для детерминированных, 20 для AI-оцениваемых — мотивирует осмысленные ответы

## 3. AI-оценка ответов (Evaluation)

**Файл:** `ai-prompts.cjs` → `EVAL_SYSTEM_PROMPTS`
**Эндпоинт:** `POST /api/evaluate`
**Модель:** `openai/gpt-4o-mini` (SSE-стриминг)

Отдельный системный промпт для каждого AI-оцениваемого типа:

### 3.1 free-response
Оценивает развёрнутый ответ. Возвращает `score` (0–100), `isCorrect` (score ≥ 60), `feedback`, `strengths`, `improvements`. Засчитывает правильную мысль даже при нестандартной формулировке.

### 3.2 explain-like-im-five
Оценивает доступность объяснения для непрофессионала. Штрафует за жаргон, поощряет аналогии. Высокий балл = объяснение понятно без знания темы.

### 3.3 teach-back
Оценивает полноту и правильность объяснения «другу». Проверяет и фактическую корректность, и понятность. Учитывает `forbiddenTerms` из задания.

### 3.4 give-your-example
Оценивает конкретность и релевантность придуманного примера. Поощряет оригинальность из реальной жизни.

### 3.5 debug-the-logic
Оценивает точность определения логической ошибки. Засчитывает, если суть ошибки понята, даже при другой формулировке. Использует `errorLocation` из задания как ориентир.

## 4. RAG-поиск (Retrieval-Augmented Generation)

**Файл:** `ai-prompts.cjs` → `RAG_SEARCH_PROMPT`
**Эндпоинт:** `POST /api/rag/search`
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Отвечает на вопрос пользователя, опираясь ТОЛЬКО на предоставленные фрагменты из базы знаний. Честно сообщает, если информации недостаточно.

**Ключевые решения:**
- «Опираясь ТОЛЬКО на предоставленные фрагменты» — предотвращает галлюцинации вне контекста платформы
- 3–7 предложений — баланс полноты и краткости
- Разрешён короткий список — для структурированных ответов на сложные вопросы

## 5. Анализ прогресса

**Файл:** `ai-prompts.cjs` → `ANALYZE_PROGRESS_PROMPT`
**Эндпоинт:** `POST /api/ai/analyze-progress`
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Анализирует историю прохождений студента и выявляет конкретные слабые места. Возвращает `summary`, `weakAreas[]` (тема + проблема + совет), `strengths`, `nextFocus`.

**Ключевые решения:**
- 2–4 `weakAreas`: не подавляет, но конкретен
- «Если слабых тем нет — всё равно найди что можно улучшить» — промпт всегда полезен
- Контекст передаётся как сводка `buildUserContextSummary()`, а не сырые данные — экономит токены

## 6. Рекомендации по обучению

**Файл:** `ai-prompts.cjs` → `RECOMMEND_NEXT_PROMPT`
**Эндпоинт:** `POST /api/ai/recommend-next`
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Предлагает 3 конкретные темы для следующего изучения на основе пройденных тем и слабых мест. Возвращает `recommendations[]` (название + обоснование + сложность + время) и `advice`.

**Ключевые решения:**
- Ровно 3 рекомендации — не перегружает выбором
- `estimatedMinutes` 15–60 мин — реалистичная оценка, помогает планировать
- CTA-кнопка передаёт тему в `/journey/new?topic=` — рекомендация кликабельна

---

## 7. Объяснение «попроще»

**Файл:** `ai-prompts.cjs` → `EXPLAIN_SIMPLER_PROMPT`
**Эндпоинт:** `POST /api/ai/explain`
**Модель:** `openai/gpt-4o-mini`

**Назначение:** Переписывает сложный текст (чекпоинт или теоретический блок) доступным языком. Возвращает `explanation`, `keyPoints[]` и `analogy`.

**Ключевые решения:**
- 3 `keyPoints` — самое важное, не больше
- `analogy` обязательна — якорная точка для запоминания
- Используется в двух контекстах: CheckpointExplainer на journey-page и кнопка «Объясни проще» в article reader

## Общая стратегия промптинга

| Принцип | Применение |
|---|---|
| Явный формат вывода | Каждый промпт содержит JSON-схему — исключает парсинг свободного текста |
| Языковое ограничение | «Пиши на русском языке» — консистентный UX |
| Конкретные правила | Числовые границы (2–4, 3–7) вместо «нескольких» |
| Опора на данные | Оценочные промпты ссылаются на поля задания (`errorLocation`, `forbiddenTerms`) |
| Токен-экономия | Контекст пользователя сжимается в `buildUserContextSummary()` перед отправкой |
