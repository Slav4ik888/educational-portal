# AI-расширение образовательной платформы

## Проблема
За предыдущие модули ты создал образовательную платформу:

Модуль 1: статьи + интерактивные компоненты тестирования
Модуль 2: Knowledge Journey — генерация и прохождение путешествий
Но эта платформа работает с заготовленным контентом. Статьи написаны заранее. Journey генерируются по запросу, но каждый раз с нуля. Нет памяти, нет персонализации, нет умного поиска.

Это как библиотека без библиотекаря — книги есть, но найти нужное сложно, и никто не помнит, что ты уже читал.

## Задача
Превратить платформу из "набора контента" в умную обучающую систему с AI.

Добавить:

RAG для поиска по контенту — "что я знаю о галлюцинациях?" → осмысленный ответ с источниками
AI-функциональность — не просто чат, а конкретные действия (генерация, рекомендации, анализ)
Персональный контекст — система помнит твой прогресс и адаптируется

## Процесс разработки
- Dev Notes (обязательно)
Веди DEV_NOTES.md с документацией процесса: решения, проблемы, находки.

- Итеративность
Начни с одной AI-фичи, доведи до работающего состояния, потом добавляй следующую.

- Промпт-журнал
Сохраняй промпты для RAG, генерации контента, персонализации.

## Ключевые проблемы для решения
1. RAG по контенту платформы
У тебя уже есть контент: статьи, journey, результаты тестов. Нужно сделать его "searchable" через AI.

Подумай:
- Как индексировать статьи (по секциям? по абзацам?)
- Как индексировать journey (структурированные данные, не просто текст)
- Как индексировать вопросы и ответы из тестов
- Как обновлять индекс при добавлении контента?

2. AI-функциональность (не просто чат)
"Спроси платформу" — это базово. Нужны конкретные действия:

Идеи:
- Генератор контента — создать новый Journey/тест по теме
- Анализатор прогресса — "в чём мои слабые места?"
- Рекомендатор — "что мне изучить дальше?"
- Объяснятор — "объясни мне X простыми словами"
- Связыватель — "как X связан с Y?"

Подумай:
- Какие 2-3 AI-действия самые ценные?
- Как они интегрируются в существующий UI?
- Как пользователь запускает эти действия?

3. Персональный контекст
Система должна знать:
- Что ты уже прошёл
- Где ошибался
- Сколько времени потратил
- Какие темы сильные/слабые

Подумай:
- Какие данные хранить?
- Как использовать в промптах (без переполнения контекста)?
- Как показывать пользователю его прогресс?
- Privacy — пользователь контролирует свои данные?

4. Интеграция в существующий UI
AI-фичи должны быть органичной частью платформы.

Подумай:
- Где живёт чат/поиск? (sidebar? отдельная страница? floating button?)
- Как запускать AI-действия? (кнопки? команды? контекстное меню?)
- Как показывать результаты AI? (inline? модалка? новая страница?)

5. Генерация контента в системе
В Модуле 2 ты генерировал journey по введённому тексту. Теперь — по теме, с использованием знаний платформы.

Подумай:
- Как комбинировать существующий контент с новым?
- Как сохранять сгенерированный контент?
- Как отличать "написанное человеком" от "сгенерированного AI"?

## Нужно придумать
- Стратегию индексации — как превратить контент платформы в knowledge base
- AI-функциональность — какие конкретные действия выполняет AI (минимум 2)
- Модель персонализации — какие данные хранить, как использовать
- UX интеграции — как AI-фичи вписываются в существующую платформу
- Dev Notes — документация процесса разработки

## Что сдаётся
- Всё из Модулей 1-2 (статьи на главной и journey-путешествие)
- RAG по контенту
- Минимум 2 AI-функции (не просто чат)
- Персональный контекст
- README.md — описание проекта в свободной форме:
    - Какие технологии использовал и почему
    - Как устроена система (RAG, AI-функции, персонализация)
    - Какие компоненты создал, как они работают вместе
    - Как развернуть проект локально (шаги)
    - Как попробовать: поиск, AI-действия, персонализация — что открыть, что должно произойти

## Критерии оценки
- RAG: поиск по контенту работает, источники показываются
- AI-функциональность: минимум 2 полезных действия (не просто чат)
- Персонализация: система помнит прогресс и адаптируется
- Интеграция: AI-фичи органично вписаны в платформу
- Dev Notes: понятна история разработки
- Итеративность: видно развитие проекта
